# Dharmiq v0.2 — Implementation Phases & Smoke Tests

**Status:** Completed — all phases (P0–P12) implemented.  
**Purpose:** Step-by-step playbook used to implement v0.2 incrementally. Retained as a reference for architecture and smoke-test commands.  
**Parent doc:** [`v0.2-prd-trd.md`](./v0.2-prd-trd.md)  
**Baseline:** v0.1 — synchronous chat pipeline, no SSE, no LangGraph.

> **Using v0.2 today:** set `DHARMIQ_AGENT_GRAPH_V2=true` in `.env`, run migrations (`alembic upgrade head`), bootstrap LangGraph checkpoints (see [README](../../README.md)), and start a Celery worker. Production setup is in [deployment.md](../deployment.md).

## How to use this document (historical reference)

This playbook was used to implement v0.2 phase-by-phase. It remains useful for smoke-test commands and architecture context. For day-to-day setup, use the [README](../../README.md) instead.

0. **Read the binding decisions first.** `v0.2-prd-trd.md` → "Clarification summary (Round 4 — implementation hardening)" (R4-1 … R4-12) overrides any conflicting detail. Key ones repeated inline below.
1. **One phase at a time.** Do not start phase N+1 until all smoke tests for phase N pass.
2. **Preserve v0.1 behavior** behind feature flag `agent_graph_v2` (env `DHARMIQ_AGENT_GRAPH_V2=true`) — v0.1 remains available when the flag is off.
3. **Match existing conventions:** async SQLAlchemy, Pydantic v2, pytest-asyncio, structlog, YAML prompts in `backend/dharmiq/llm/prompts/`.
4. **Every phase ends with:** code + migration (if any) + tests + smoke command block.
5. **Do not commit** unless the user asks.

### Non-negotiable gotchas (from Round 4)

- **R4-5:** never declare a model attribute named `metadata`. Use `chunk_metadata = mapped_column("metadata", JSONB, ...)`.
- **R4-1:** models are `deepseek/deepseek-v4-flash` (fast) and `deepseek/deepseek-v4-pro` (answerer + validator-with-reasoning). `deepseek-chat` is deprecated.
- **R4-2:** embeddings stay local 384-dim; not config-swappable. LiteLLM = chat always + remote embed/rerank only.
- **R4-3:** clarifier is END-and-return; round counter lives in `chat_requests.clarifier_round`.
- **R4-4:** streaming = replay the validated string; no second LLM call.
- **R4-6:** `search_vector` is a GENERATED STORED tsvector column + GIN; no trigger.
- **R4-7:** event `seq` via Redis `INCR`, not `max(seq)+1`.

### Config snippet

```yaml
# config/config.dev.yaml
agent_graph:
  enabled: false   # overridden by DHARMIQ_AGENT_GRAPH_V2=true

llm:
  roles:
    primary: openrouter/deepseek/deepseek-v4-pro     # answerer (R4-1)
    fast: openrouter/deepseek/deepseek-v4-flash      # aux agents (NOT deepseek-chat — deprecated)
    embedding: local                                 # FIXED 384-dim; not config-swappable (R4-2)

    local_model: BAAI/bge-reranker-base
    litellm_model: cohere/rerank-english-v3.0
```

> **Embedding note (R4-2):** `embedding: local` is fixed for v0.2. Do not wire `aembedding`/remote embedding swap — it would require a column-dimension migration + full re-embed.

### Graph nodes (minimum for P2)

```text
input_guard → clarifier → [END + return followups if needs_clarification AND round<3 AND not force_answer]
           → query_rewriter → retrieve (merged, v0.1 retrieval for now)
           → answerer → validator → [regen loop ≤3] → finalizer
```

Clarifier is END-and-return (R4-3): no in-graph wait. Round 4+ or `force_answer` proceeds with `stated_assumptions`.

## P6 — Parent-child chunking + reindex

### Goal

Section-aware parent/child chunks; embed children; answer with parent text.

### Tasks

| # | Task | Files |
|---|------|-------|
| 6.1 | `chunker.py`: emit parent + child chunks with `parent_chunk_id`; **token counts via the MiniLM tokenizer** (`sentence-transformers` model tokenizer), not chars (v0.1 used chars) | `backend/dharmiq/ingestion/chunker.py` |
| 6.2 | `context_text` generator at ingest (≤512 tokens) | `backend/dharmiq/ingestion/context_text.py` |
| 6.3 | Update corpus + upload pipelines | `ingestion/pipeline.py`, `upload_pipeline.py` |
| 6.4 | Celery backfill task `reindex_corpus_v02` — **non-destructive**: write new chunks alongside old, flip `is_active`/soft-delete old only after new set verified (keeps search live during reindex) | `backend/dharmiq/tasks/ingestion_tasks.py` |
| 6.5 | Retrieval: child search → parent hydration via `parent_chunk_id` (R4 / B11.4) | `retrieval/hybrid.py` |

### Reindex runbook (R4 — P6 is heavy/destructive)

- This re-chunks (200–400 **token** children vs v0.1's 2–4 KB **char** chunks) and **re-embeds the entire corpus + all uploads**. Estimate chunk count and embed time on the target VPS before running; record it.
- Use the dual-write strategy (6.4) so live search is not degraded mid-rebuild.
- Tokenizer is the **local MiniLM tokenizer** (matches the fixed embedding model, R4-2) — document the exact `child_chunk_target_tokens` counting method in `chunker.py`.

## Risk register (agent gotchas)

| Risk | Mitigation |
|------|------------|
| `metadata` reserved attribute crash (R4-5) | Map column via `chunk_metadata = mapped_column("metadata", ...)`; `test_chunk_metadata_attr_maps_to_metadata_col` |
| Invalid/deprecated model string (`deepseek-chat`) | Use `deepseek-v4-flash`/`-pro` (R4-1); centralize in config; mock-test |
| Embedding swap assumed config-only (R4-2) | Doc: embeddings fixed 384-dim; swap = migration + full re-embed |
| Quote substring mismatch → regen storm (R4-8) | Normalized fuzzy matcher; `test_quote_fuzzy_match_*` |
| Event `seq` race under parallel nodes (R4-7) | Redis `INCR`; unique constraint backstop; `test_seq_no_collision_under_parallel_emit` |
| Double LLM cost from re-streaming (R4-4) | Replay validated string; `test_stream_is_replay_not_second_call` |
| SSE reconnect gap (pub/sub) | Subscribe → replay `>after_seq` → dedupe; `test_reconnect_replays_then_dedupes` |
| Heavy P6 reindex degrades search | Non-destructive dual-write; reindex runbook |
| SSE proxy buffering | nginx `proxy_buffering off` for `/stream` — document in deployment.md |
| Cross-encoder RAM on VPS | Lazy-load reranker; singleton; confirm VPS RAM budget |
| Version drift (`langgraph`/checkpoint/`litellm`) | **Pin exact / `~=`** (R4-10); lock in `uv.lock` |
| LangGraph checkpoint bootstrap outside Alembic | One-time idempotent `AsyncPostgresSaver.setup()`; doc in deployment.md |
| Breaking v0.1 chat API | Feature flag + parity test `test_chat_pipeline` |
| Debug leak (live + replay) | Server-side strip (R4-9); integration test with non-admin JWT on both paths |
